# 声形绘 UI 设计详述

> 设计概念：**白昼纸面，暗夜雷电**
> 静态界面 = 暖白纸面 + 墨黑衬线字（极简现代风）
> 演奏瞬间 = 纯黑画布 + 霓虹雷电光晕（赛博朋克感）
> 这个强对比是产品的视觉记忆点

---

## 一、设计概念

### 1.1 核心隐喻

整个产品的视觉叙事围绕一个对比：

- **白昼**（非演奏态）：暖白纸面、墨黑文字、衬线大标题、大量留白。像翻开一本设计精良的书。安静、克制、优雅。
- **暗夜**（演奏态）：当摄像头开启、手部追踪激活的瞬间，画面切换为纯黑画布，虚拟键位像雷电一样在黑暗中亮起，手指触碰时迸发霓虹光晕。像在夜空中弹奏闪电。

这个对比让用户每次开始演奏都有"仪式感"——从平静的纸面进入雷电交加的演奏场。

### 1.2 一句话记忆点

> "翻开是纸，弹响是电。"

## 二、配色系统

### 2.1 白昼纸面（静态界面）

```css
/* 背景 —— 不是纯白，是带暖意的纸色 */
--paper-bg: #FAFAF7;          /* 主背景，像宣纸/象牙白 */
--paper-bg-elevated: #FFFFFF; /* 卡片背景，纯白 */
--paper-bg-recessed: #F0EFEA; /* 凹陷区域，更深一档 */

/* 文字 */
--ink-primary: #0A0A0A;       /* 墨黑，主标题 */
--ink-secondary: #4A4A4A;     /* 次级文字 */
--ink-tertiary: #9A9A9A;      /* 辅助/占位 */

/* 强调色 —— 只在关键操作出现 */
--accent-electric: #0066FF;   /* 电蓝，主按钮 */
--accent-electric-deep: #0044CC; /* hover 加深 */
--accent-success: #00B86B;    /* 识别成功 */
--accent-warn: #FF8800;       /* 调音偏差 */
--accent-error: #E63229;      /* 识别失败 */

/* 分割线 */
--hairline: #E5E3DD;          /* 细发丝线，像铅笔痕 */
```

### 2.2 暗夜雷电（演奏区）

```css
/* 背景 */
--night-bg: #000000;          /* 纯黑画布 */

/* 霓虹三原色 —— 雷电感 */
--neon-cyan: #00F0FF;         /* 青色，主光晕 */
--neon-magenta: #FF00C8;      /* 品红，次光晕 */
--neon-yellow: #FFE600;       /* 黄色，高亮提示 */

/* 键位状态 */
--key-idle: rgba(0, 240, 255, 0.15);    /* 默认：低透明青色描边 */
--key-hover: rgba(0, 240, 255, 0.6);    /* 手指悬停：描边加亮 */
--key-triggered: #00F0FF;               /* 触发瞬间：满色 */

/* 手部追踪 */
--hand-glow: rgba(255, 255, 255, 0.9);  /* 指尖发光点 */
--hand-trail: rgba(255, 255, 255, 0.4); /* 拖尾粒子 */
```

### 2.3 颜色使用原则

- 白昼界面：90% 是纸色+墨色，电蓝只出现在主按钮和焦点状态
- 暗夜界面：黑色为主，霓虹色只在触发瞬间爆发，平时克制
- **绝不用紫色渐变**（AI 通用美学的大忌）

## 三、字体系统

### 3.1 字体选择

```css
/* 标题字体：Fraunces —— 现代衬线，有个性不雷同 */
--font-display: 'Fraunces', 'Source Han Serif SC', serif;

/* 正文字体：Inter Tight —— 紧凑无衬线，比 Inter 更有性格 */
--font-body: 'Inter Tight', -apple-system, 'PingFang SC', sans-serif;

/* 等宽字体：JetBrains Mono —— 用于音名、频率显示 */
--font-mono: 'JetBrains Mono', 'SF Mono', monospace;
```

**为什么选 Fraunces 而不是 SF Pro：**
- Fraunces 是可变字体，字重和光学尺寸丰富
- 衬线结构传递"纸面/书写"的隐喻，契合"画出来就能弹"的主题
- 比 Apple SF Pro 更有辨识度，避免做成"又一个苹果仿品"

### 3.2 字号阶梯

```css
--text-hero: 88px;     /* 首页巨标题，字重 300（细衬线巨字） */
--text-h1: 56px;       /* 页面主标题 */
--text-h2: 36px;       /* 区块标题 */
--text-h3: 24px;       /* 卡片标题 */
--text-body: 17px;     /* 正文 */
--text-caption: 14px;  /* 辅助 */
--text-micro: 11px;    /* 标签，全大写字母间距加大 */

/* 字重 */
--weight-light: 300;   /* 巨标题用，优雅 */
--weight-regular: 400;
--weight-medium: 500;
--weight-semibold: 600;
--weight-bold: 700;
```

### 3.3 字体使用细节

- 巨标题用 Fraunces Light（300），字间距 -0.02em，行高 1.05
- 正文用 Inter Tight Regular，行高 1.6
- 音名/频率用 JetBrains Mono，全大写，字间距 0.05em
- 微标签（如"STEP 01"）用 Inter Tight Medium，全大写，字间距 0.15em

## 四、动画系统

### 4.1 动画原则

- **白昼克制**：静态界面动画微妙、缓慢、优雅（200-400ms）
- **暗夜爆发**：演奏区动画快速、强烈、有冲击力（80-200ms）
- **节奏对比**：平静的进入 → 爆发的触发 → 缓慢的衰减

### 4.2 页面级动画

**首页加载（staggered reveal）**
```
0ms    背景纸色淡入（400ms）
200ms  顶部导航从上滑入（300ms，ease-out）
400ms  巨标题"画出来，就能弹。"逐字淡入（每字间隔 50ms）
800ms  副标题从下淡入（300ms）
1000ms 两个按钮从下淡入 + 上移 8px（400ms）
1200ms 演示动图从下淡入 + 模糊到清晰（500ms）
```

**页面切换**
- 淡出 150ms → 淡入 200ms
- 不用滑动，保持纸面的"翻页感"

**滚动揭示**
- 卡片进入视口时：从下淡入 + 上移 16px（400ms，ease-out）
- staggered：相邻卡片延迟 80ms

### 4.3 组件级动画

**按钮 hover**
```css
/* 主按钮 */
transform: translateY(-2px);
box-shadow: 0 8px 24px rgba(0, 102, 255, 0.25);
transition: all 200ms cubic-bezier(0.4, 0, 0.2, 1);

/* 点击瞬间 */
transform: scale(0.97);
transition: transform 80ms;
```

**卡片 hover**
```css
transform: translateY(-4px);
box-shadow: 0 16px 48px rgba(0, 0, 0, 0.08);
transition: all 300ms cubic-bezier(0.4, 0, 0.2, 1);
```

**输入框 focus**
```css
border-color: #0066FF;
box-shadow: 0 0 0 4px rgba(0, 102, 255, 0.1);
transition: all 200ms;
```

### 4.4 演奏区特效动画（核心记忆点）

**摄像头开启瞬间（白昼→暗夜切换）**
```
0ms    演奏区从中心圆形扩散变黑（clip-path: circle，500ms）
300ms  虚拟键位从中心向外逐个亮起（staggered，每键延迟 30ms）
600ms  "请把手放在画面内"提示淡入
```

**手部追踪持续动画**
- 指尖发光点：白色圆点 + 外圈光晕，跟随手部移动（无延迟）
- 拖尾粒子：每帧生成 1 个粒子，渐隐 400ms，位置略微随机偏移
- 手部骨架连线：半透明白色（rgba(255,255,255,0.3)），1px 细线

**键位触发瞬间（最重要！）**
```
0ms     手指进入键位区域
10ms    键位描边从 0.15 透明度 → 满色（80ms）
20ms    从触点爆发粒子（12 个粒子，径向扩散，400ms 渐隐）
40ms    光晕从触点圆形扩散（半径 0 → 80px，300ms）
60ms    音色发声（Web Audio）
100ms   键位内部填充满色（100ms）
400ms   光晕开始渐隐（600ms）
800ms   键位回到默认状态（200ms）
```

**键位悬停（未触发）**
- 描边透明度 0.15 → 0.6（150ms）
- 内部微微发光（radial-gradient，从中心 0.1 透明到边缘 0）

**音符衰减**
- 键位颜色饱和度从 100% → 30%（与音频 release 同步，500ms）

### 4.5 调音器指针动画

- 指针位置：smooth 过渡（200ms ease-out）
- 偏差越大，指针颜色越红（从青色 → 黄色 → 红色渐变）
- 居中（准音）时：指针变绿 + 微微脉动（scale 1 → 1.1 → 1，400ms）

### 4.6 加载状态

- 生成键位中：纸面上显示一个墨水滴落的动画（SVG，循环 1.2s）
- 识别中：旋转的细线圆环（不是 spinner，是优雅的墨迹圆环）
- 摄像头初始化：黑色画面 + 中心白色光点扩散

## 六、微交互细节

### 6.1 画布交互

- 画笔光标：自定义十字准星（不是默认箭头）
- 画线时：墨迹效果（线条边缘略有扩散，像毛笔）
- 画方块：拖拽时显示尺寸标注（如"120 × 60"）
- 撤销：画布轻微闪白（50ms）+ 上一步形状淡出

### 6.2 示例模板下拉

- 点击"示例模板 ▾"：下拉菜单从上淡入（150ms）
- 每个模板显示：缩略图 + 名称（如"标准钢琴 8 键"）
- 选中后：模板形状逐个画到画布上（staggered，每形状 100ms）

### 6.3 摄像头开启过渡

- 点击"生成键位并开始演奏"：
  1. 按钮缩放反馈（80ms）
  2. 画布淡出（200ms）
  3. 演奏区从中心圆形扩散变黑（clip-path，500ms）
  4. 虚拟键位逐个亮起（staggered，每键 30ms）
  5. "请把手放在画面内"提示淡入（300ms）

### 6.4 触发反馈层次

每次键位触发有 5 层反馈叠加：
1. **听觉**：Web Audio 发声（10ms 内）
2. **视觉-键位**：描边满色 + 内部填充（80ms）
3. **视觉-光晕**：从触点圆形扩散（300ms）
4. **视觉-粒子**：12 个粒子径向爆发（400ms）
5. **视觉-手部**：指尖爆发额外粒子（200ms）

### 6.5 错误状态

- Toast 提示：从顶部下滑（300ms），3 秒后上滑消失
- 错误卡片：红色左边框（4px）+ 浅红背景
- 摄像头授权失败：演奏区显示引导图 + "开启权限"按钮

### 6.6 空状态

- 无演奏记录：显示"还没有演奏记录，去工作台试试 →"
- 未登录访问个人中心：引导登录卡片

## 八、性能优化

- 演奏区 Canvas 用 `requestAnimationFrame`，禁用 CSS transition
- 手部追踪 30fps，特效渲染 60fps（插值平滑）
- 粒子数量上限 200，超出自动回收最老的
- 移动端自动降低：粒子数减半、光晕半径减小
- 帧率 < 20fps 时：自动关闭粒子，保留光晕

## 九、无障碍

- 所有按钮有 `aria-label`
- 颜色对比度：墨黑/纸白 ≥ 15:1，电蓝/纸白 ≥ 4.5:1
- 键盘可导航：Tab 焦点环用电蓝
- 演奏区有"仅键盘模式"：用电脑键盘按键代替手部追踪（备用方案）
- `prefers-reduced-motion`：关闭粒子和大动画，保留必要反馈
